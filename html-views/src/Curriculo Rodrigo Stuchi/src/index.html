Projeto: Cartão de Senhas a proposta inicial era - todos os clientes terão um cartão de senhas. Desenvolvi a primeira prova de conceito gerando cartões para todos os clientes, levou mais de 1 hora para finalizar. Trabalho pesado de criptografia, o gargalo era má utilização do processador. A segunda PoC utilizei a TPL (Task Parallel Library), com Thread-Safe Collections. Com a TPL o processamento era concluído em apenas 15 minutos, 400% mais rápido.
Esse serviço contempla todos os processos de gestão de cartões, geração dos tokens, validação das senhas por posição, bloqueio da posição por exceço de tentativas, controle de extravio, contagem de uso, envio de arquivo para gráfica com tolerância a falhas, quando há falha no envio novas tentativas são feitas, excedendo as tentativas, os lotes do dia são enviados no dia seguinte. Parametrizável e 100% auto gerenciável.

Projeto: SystemCache sistema de cache inteligente utiliza a biblioteca MemoryCache do .Net faz cache de arquivos e consultas no SQL Server, quando alguma informação no banco ou arquivo é alterada, o cache é invalidado e atualizado na próxima consulta. Utiliza o SQL Server Service Broker, que envia eventos para a aplicação invalidar o cache, para arquivos utiliza o FileSystemWatcher com a mesma lógica.
Os testes com o SystemCache foram conclusivos, 100.000 consultas (tabela com 30 registros, 3 colunas, com where).

Resultado ~12 minutos sem cache, utilizando o SystemCache foram apenas 0.6 segundos.

Utilizando Gulp as dependências (css / js) são injetadas, imagens convertidas para base64, o conteúdo fixo (hardcoded) é transformado em conteúdo dinâmico (razor) por expressão regular.
